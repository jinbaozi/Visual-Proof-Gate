# Progressive Disclosure Refactor Execution

## Slices executed

1. Contracts: primitives, config, evidence, defect, viewport, stage, and context contracts.
2. Runtime setup: config loader/validator, intent renderer, IO helpers, browser adapters.
3. Probes: layout, responsive, content stress, assets, tokens, states.
4. Reports and routing: Markdown renderers and Impeccable command plan.
5. Orchestrator: explicit `VISUAL_PROOF_STAGES` and generic stage runner.
6. Guardrails: architecture tests and CI integration.

## Architectural effect

The former monolithic implementation is now a compatibility re-export. Agents can navigate from README and `agent-map.md` to the narrow layer needed for a task.
